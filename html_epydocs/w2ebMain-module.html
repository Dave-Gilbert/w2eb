MainRecursiveCall(opts, bl, url, footnote, foot_title, foot_dict_list)

source code 

Fetches footnotes and subarticles by calling main function

Parameters:
  • opts - Dictionary of common CLI parameters. See StartupGetOptions()
  • bl - Beautiful Soup representation of an HTML web page.
  • url - Url to download footnotes / new sections from
  • footnote - Boolean, whether or not to generate a footnote
  • foot_title - The name of the footnote
  • foot_dict_list - - a list of footnotes already processed
Returns:
(err - any errors encountered as a string, foot_dict - footnote data stored in a dictionary)

Note: Subsections are returned implicitly by collecting data in bodir. Notice that we make a new 'bodir' for this invocation that is the current bodir / basename(url)

For footnotes, turning on the footnote flag causes us to write output in the current directories "footnote" directory. We also return the same data in a dictionary structure from main.

MainGetHref(opts, foot_dict_list, anch, footnote, foot_title, foot_dict, outdir, sect_dict, url)

source code 

Return the href for footnotes and sections

Parameters:
  • opts - Dictionary of common CLI parameters. See StartupGetOptions()
  • foot_dict_list - List of footnotes represented by dictionaries
  • anch - the html anchor for defining jumplinks in id=<anch>
  • footnote - Boolean, whether or not to generate a footnote
  • foot_title - The name or string reference to a footnote
  • foot_dict - Dictionary storing details for notes and footnotes
  • outdir - The output directory that .html files are written to
  • sect_dict - Dictionary of items describing a section
  • url - Url to download footnotes / new sections from
Returns:
(href - the html field for href=

Note: At this stage we have either found the reference as a footnote, or a full section. Whether it was in the cache or not doesn't matter at this stage. We should be able to complete the href.

The special case for depth = 0 means that we definitely did not download new material. At depth = 0 we might have found a cached reference, but if not we return the URL as the href.

If we don't have an href by the end of this fn there is a bug.

Decide whether the tag will be processed as a section or a footnote
